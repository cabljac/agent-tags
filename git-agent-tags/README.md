# git-agent-tags

A git subcommand for managing `@agents` tags in codebases. Parses tags, builds a reference graph, detects stale headers via git history, and outputs scoped context for AI agents.

## Install

```bash
cargo install --path .
```

The binary is named `git-agent-tags` so it runs as `git agent-tags` from any git repo.

## Commands

```bash
# Print all @agents tags to stdout (for agents to read before touching code)
git agent-tags context
git agent-tags context > context.md

# Scoped context: only tags reachable from a specific file
git agent-tags context --for src/auth/token.ts
git agent-tags context --for src/auth/token.ts --hops 2

# Overall health: files scanned, headers present, broken refs
git agent-tags status

# Show stale headers and broken references
git agent-tags check
git agent-tags check --deep   # also runs regex heuristics

# Show only broken Related: references (renamed or deleted files)
git agent-tags broken

# List files with no @agents header
git agent-tags missing

# Suggest Related: links based on co-change history
git agent-tags suggest

# Show the reference graph for a specific file
git agent-tags graph src/auth/token.ts

# Rebuild the file index cache
git agent-tags reindex

# Run as a pre-commit hook (fail on broken refs, warn on staleness)
git agent-tags hook
git agent-tags hook --deep   # also runs regex heuristics

# Install the pre-commit hook into .git/hooks
git agent-tags hook --install
```

## The context command

`git agent-tags context` scans the entire repo for every `@agents` annotation — both file headers and inline tags — and prints them to stdout. An AI agent can call it directly and read the output before touching any code.

```markdown
# agent-context
<!-- Generated by git-agent-tags. Do not edit manually. -->

## src/auth/token-refresh.ts#auth-module
Handles silent refresh. httpOnly cookies only.
Related: src/auth/cookie-config.ts

## src/firestore/useDocumentQuery.ts:34
Query key must include documentRef.path, not just id.

## src/mutations/invalidate.ts:12#pre-mutation
Must invalidate BEFORE mutation callback, not after.
```

- File headers: `## path/file.ts` or `## path/file.ts#name`
- Inline tags: `## path/file.ts:LINE` or `## path/file.ts:LINE#name`
- Sorted by file path, then line number

### Scoped context with `--for`

`--for <file>` walks the reference graph and only outputs tags from files reachable from the given file. This gives an AI agent exactly the context it needs when editing a specific file, without flooding the context window.

`--hops N` controls the walk depth (default: 1). Both outgoing (`Related:`) and incoming references are followed.

```bash
# Tags from token.ts + all files it references or is referenced by
git agent-tags context --for src/auth/token.ts

# Go deeper: 2 hops of the graph
git agent-tags context --for src/auth/token.ts --hops 2

# Just the file's own tags
git agent-tags context --for src/auth/token.ts --hops 0
```

## Pre-commit hook

`git agent-tags hook --install` adds a pre-commit hook to `.git/hooks/`. On each commit:

- **Broken references** (renamed/deleted files, invalid fragments) → **block the commit** (exit 1)
- **Stale headers** → **print warnings** but allow the commit (exit 0)

If a pre-commit hook already exists, the agent-tags check is appended to it. If `git-agent-tags` isn't installed, the hook silently skips.

## Staleness detection

Two tiers, run in order of cost:

1. **Git heuristics** — compare the last commit that touched the `@agents` block (via `git blame`) against the last commit that touched the file. If the gap is large or more than 40% of lines changed, the header is flagged stale.
2. **Regex heuristics** (`--deep`) — scan the diff since the header was last updated for new exports or imports not mentioned in `Related:`.

### Scoped staleness with `lines_owned`

Tags can declare how many lines of code they own:

```ts
// @agents(token-check, 15): Must validate before refresh.
const isValid = checkToken(token);  // ← line 1 of 15
```

When `lines_owned` is set, staleness is checked only against the owned range (the N lines after the comment ends) rather than the whole file. This avoids false positives when unrelated parts of a large file change.

## Configuration

Optional `.git-agent-tags.toml` at the repo root:

```toml
# Additive — merged with the built-in defaults, not a replacement.
# Built-in defaults include: node_modules, dist, build, target, .git,
# *.md, *.mdx, *.rst, *.txt, *.lock, *.sum, images, fonts, *.min.js, *.map
ignore = ["*.test.ts", "*.spec.ts", "fixtures"]

stale_commit_gap = 10       # commits before flagging stale
stale_diff_percent = 40     # % of lines changed before flagging stale

cochange_min_commits = 3    # min co-changes before suggesting Related:
cochange_max_files = 20     # ignore large bulk commits
```

## Reference graph

The tool builds a directed graph from `Related:` and `See:` links. The `broken` command finds edges pointing to files that no longer exist or fragments that don't resolve. The `graph` command shows incoming and outgoing links for any file.

The graph is cached in `.git/agent-tags/index.json` (not committed).
